ScopeGuard — Data Model & State Machine

Upstash Redis schema and project lifecycle

Project State Machine

DRAFT
→
Freelancer reviews SOW
→
PENDING
→
Client approves & locks scope
→
LOCKED
→
Freelancer marks complete
→
COMPLETED

Note: Added DRAFT state — SOW is generated but freelancer hasn't shared the link yet. This separates "AI generated it" from "client can see it."

Redis Key Structure

project:{projectId} → full project JSON object
// That's it. One key per project. Redis handles the rest.

Key Design Decisions

DRAFT state added

Freelancer can review/edit the AI-generated SOW before sharing with client

scopeVerdict on log entries

Each client request carries its AI verdict + optional freelancer review status

aiReasoning stored

Transparency — both parties can see why the AI flagged something

surcharge field

Freelancer can attach a cost when approving gray-area / out-of-scope requests
