ScopeGuard — Architecture Overview

Page structure, API routes, and data flow

Pages

/create
Freelancer Setup
  • Paste client notes
  • Set base price
  • Add Raenest link
  • AI generates SOW
  • Review & generate portal link
/project/[id]
Freelancer Dashboard
  • View SOW & status
  • Post updates to timeline
  • Review gray-area requests
  • Mark project complete
  • Share client portal link
/portal/[id]
Client Portal
  • View SOW (read-only)
  • Approve & lock scope
  • Submit change requests
  • View activity timeline
  • Release payment button

API Routes

POST /api/generate-sow

Takes client notes + price → calls Mistral → returns SOW JSON → saves to Upstash Redis

POST /api/approve

Client approves SOW → status changes PENDING → LOCKED

POST /api/update

Freelancer posts update or marks complete → appends to activity log

POST /api/check-scope

Client submits request → Mistral compares against locked SOW → returns IN_SCOPE / GRAY_AREA / OUT_OF_SCOPE

POST /api/review

Freelancer approves/rejects gray-area request → updates log entry + optional surcharge

GET /api/project/[id]

Fetches full project JSON from Upstash Redis — used by all pages

Data Flow

Freelancer → pastes notes → /api/generate-sow → Mistral AI → SOW JSON → Upstash Redis
Freelancer → shares link → Client opens /portal/[id]
Client → approves → /api/approve → status: LOCKED → Upstash Redis
Client → submits request → /api/check-scope → Mistral AI → 3-tier verdict
Freelancer → marks complete → status: COMPLETED → payment button appears
